# Статистический анализ датасетов: технический отчёт

## Аннотация

В данном отчёте описан второй уровень системы обнаружения отравления обучающих данных. Блок реализует 16 математических методов статистического анализа, работающих без обученной модели и без эталонного (чистого) датасета. Приводится обоснование выбора методов, их математическое описание, обоснование пороговых значений и анализ покрытия угроз. Отдельно рассмотрены пределы применимости блока.

---

## 1. Постановка задачи

Отравление обучающих данных (data poisoning) — класс атак на ML-системы, при котором злоумышленник вносит изменения в данные до начала обучения модели. По классификации атак выделяют несколько механизмов воздействия:

- **Label flipping** — подмена меток правильных примеров с целью снизить общую точность или создать систематическую ошибку для целевого класса
- **Outlier injection** — добавление или модификация признаков для получения значений далеко за пределами нормального распределения
- **Backdoor injection** — вставка триггерных паттернов (обычно в текстовые признаки), активирующих желаемое поведение модели при наличии триггера во входных данных
- **Feature perturbation** — замена реальных значений признаков на технические плейсхолдеры или значения из другой области
- **Homoglyph substitution** — замена символов визуально идентичными Unicode-аналогами (например, кириллические 'а', 'е', 'о' вместо латинских) с целью обойти токенизацию и санитарные проверки

Задача блока статистического анализа — выявить следы этих атак на уровне распределений, структурных аномалий и межклассовой сепарабельности, не опираясь на обученную модель и не требуя чистого референсного датасета.

---

## 2. Место блока в системе

Система организована в три последовательных уровня защиты:

1. **Санитарные проверки** — детерминированная валидация: физически невозможные значения, инъекции команд и промптов
2. **Статистический анализ** — данный блок
3. **Анализ обученной модели** — активационные методы, спектр весов, функции влияния (требуют обученной нейросети)

Принципиальное преимущество блока 2: он работает до начала обучения, что позволяет предотвратить загрязнение весов. Блок 3 уже анализирует последствия; блок 2 — причины.

## 3. Архитектура блока

### 3.1 Векторизация

Все методы, работающие с расстояниями и распределениями, требуют числовой матрицы признаков. Реализован универсальный векторизатор, покрывающий три типа данных:

| Тип колонки | Критерий | Обработка |
|---|---|---|
| Числовая | `is_numeric_dtype` | Медианная импутация → StandardScaler |
| Категориальная | Строки, средняя длина < 25 символов | OrdinalEncoder → StandardScaler |
| Текстовая | Строки, средняя длина ≥ 25 символов | TF-IDF, 200 юниграмм, `sublinear_tf=True` |

Нормировка всех признаков к единому масштабу — обязательное условие корректности дистанционных методов (M8–M13), поскольку без неё признаки с разными единицами измерения создают доминирующий эффект масштаба.

Для определения целевой колонки реализован трёхуровневый эвристический поиск: точное совпадение с распространёнными именами → нормализованное совпадение по ключевым словам (exited, risk, fraud и др.) → совпадение по префиксу (например, `default.payment.next.month`).

### 3.2 Порядок исполнения

Сканеры разделены на два слоя. **Структурные сканеры** работают с сырым датафреймом и запускаются параллельно с векторизацией. **Статистические методы M1–M16** читают из матрицы признаков, которая вычисляется один раз и передаётся через контекст выполнения; если векторизация упала, они пропускаются без остановки пайплайна.

## 4. Структурные сканеры

### 4.1 Числовые сентинел-значения

**Формализация:** значение $v$ в числовой колонке является сентинелом, если выполнены оба условия:

$$\text{freq}(v) \geq 0.01 \cdot n \quad \text{И} \quad \frac{|v - Q_3|}{Q_3 - Q_1} > 3 \quad \left(\text{или} \quad \frac{|Q_1 - v|}{Q_3 - Q_1} > 3\right)$$

Строгое неравенство `>` (не `≥`) принципиально: граница Tukey для экстремальных выбросов — ровно 3 IQR. Натуральные крайние значения нередко попадают на эту границу в целочисленных колонках (стандартные сроки займа, коды задержки платежей). Инжектированные значения всегда превышают границу с запасом, поскольку выбираются из диапазона, физически недостижимого для данного признака.

**Покрываемые угрозы:** Feature perturbation, целенаправленная замена реальных значений на технические плейсхолдеры для создания специфической зависимости между триггером и целевым классом.

### 4.2 Обнаружение бэкдор-токенов

Для каждой текстовой колонки датасета вычисляется документная частота каждого токена. Токен признаётся подозрительным, если:

1. Он встречается в > 10% документов
2. Соответствует паттерну `(?:[0-9]+[a-zA-Z]+|[a-zA-Z]+[0-9]+)[a-zA-Z0-9]*` — перемежающиеся буквы и цифры, характерная сигнатура синтетических триггеров

Обработка всех текстовых колонок датасета (а не только основной) принципиальна: в датасетах для instruction-tuning триггер обычно вносится в короткую инструкционную колонку, а не в длинный текст ответа.

**Покрываемые угрозы:** Backdoor injection в текстовых датасетах для классификации намерений, анализа тональности, задач NLU.

### 4.3 Не-ASCII символы

Если доля не-ASCII символов в текстовой колонке превышает 0.5%, это является индикатором атаки подмены букв омоглифами. Атака основана на том, что человек не различает `a` (U+0061) и `а` (U+0430), тогда как токенизаторы и векторные модели обрабатывают их как полностью разные единицы.

**Покрываемые угрозы:** Homoglyph substitution — атака, направленная на изменение поведения модели при внешней сохранности текста.

### 4.4 Конфликт меток

Строки с идентичными вектором признаков, но разными метками обнаруживаются через хэширование строк pandas. Порог разбиения: до 2% конфликтующих строк — аннотационный шум (в датасетах тональности разметчики объективно расходятся во мнениях), 2% и выше — блокировка.

### 4.5 Дублированные строки

Точные дубликаты всей строки. Порог блокировки — 35%: некоторые NLP-датасеты содержат значительное число естественных дублей (одинаковые новостные заголовки с разными инструкциями, одинаковые запросы в разных контекстах). Порог выбран с запасом для исключения ложных срабатываний.

### 4.6 Баланс классов

Соотношение максимального и минимального класса > 5:1 фиксируется как HAND_CHECK, но не блокирует пайплайн. Обоснование: без референсного датасета нельзя отличить «атаку раздутия миноритарного класса» от задачи с естественным дисбалансом (фрод-детекция, редкие болезни). Жёсткая блокировка создаёт систематические ложные срабатывания на легитимных данных.

## 5. M1–M7: Одномерные статистические тесты

### M1 — Modified Z-score на основе MAD

*Iglewicz, B. & Hoaglin, D.C. (1993). How to Detect and Handle Outliers. ASQ Press.*

$$M_i = \frac{0.6745 \cdot (x_i - \tilde{x})}{\text{MAD}}, \quad \text{MAD} = \text{median}(|x_i - \tilde{x}|)$$

Порог: $|M_i| > 3.5$ (критерий из оригинальной монографии).

**Обоснование выбора MAD:** медиана и MAD обладают точкой разрушения 50% — они устойчивы к влиянию самих выбросов, тогда как стандартный Z-score на основе $\bar{x}$ и $\sigma$ «тянется» за аномалиями и теряет различающую способность именно тогда, когда она нужна.

Метод возвращает HAND_CHECK (не блокирует): PCA-преобразованные финансовые признаки имеют тяжёлые хвосты и могут показывать 5–7% выбросов в норме. Жёсткую инъекцию признаков надёжнее ловит NumericSentinelValueScanner.

### M2 — Анализ высших моментов

*Barreno, M. et al. (2010). The security of machine learning. Machine Learning, 81(2).*

Для каждой пары (признак, класс) вычисляются:

$$\text{Skewness} = \frac{E[(X - \mu)^3]}{\sigma^3}, \qquad \text{Kurtosis} = \frac{E[(X - \mu)^4]}{\sigma^4} - 3$$

Пороги (из Barreno et al.): $|\text{Skewness}| > 2$, избыточный куртозис $> 10$ (тяжёлый хвост) или $< -1.5$ (платикуртическое/двугорбое распределение).

Двугорбое распределение признаков внутри класса является следом подмены меток: класс содержит смесь двух популяций с разными статистиками.

### M3 — Тест Колмогорова–Смирнова (внутриклассовая согласованность)

*Kolmogorov, A.N. (1933). Sulla determinazione empirica di una legge di distribuzione; Ozdag et al. (2023), ICISSP.*

Каждый класс случайно делится пополам. Для каждого признака:

$$D = \sup_x |F_{\text{A}}(x) - F_{\text{B}}(x)|$$

Если класс однороден, обе половины имеют одно распределение и $D$ мал. При смешении двух источников распределений (следствие подмены меток) случайные половины будут статистически различимы по многим признакам. Применяется поправка Бонферрони на множественные сравнения.

### M5 — Дивергенция Дженсена–Шеннона

*Lin, J. (1991). Divergence measures based on the Shannon entropy. IEEE Trans. Inf. Theory, 37(1).*

$$\text{JSD}(P \| Q) = \frac{1}{2} D_{KL}(P \| M) + \frac{1}{2} D_{KL}(Q \| M), \quad M = \frac{P+Q}{2}$$

Симметричная, ограниченная ($\text{JSD} \in [0, 1]$ при основании 2), сглаженная мера расхождения. Слишком низкое среднее JSD между классами ($< 0.05$) означает, что классы неотличимы в пространстве признаков — индикатор загрязнения подменой меток.

### M6 — Расстояние Вассерштейна

*Villani, C. (2008). Optimal Transport: Old and New. Springer; Arjovsky et al. (2017). WGAN, ICML.*

$$W_1(P, Q) = \inf_{\gamma \in \Gamma(P,Q)} \int |x - y| \, d\gamma(x,y)$$

В отличие от KS и JSD, учитывает геометрию пространства. Каждый класс сравнивается с глобальным распределением признаков. Аномально высокое расстояние указывает на присутствие в классе образцов с систематически другим распределением признаков.

### M7 — Критерий χ² для категориальных признаков

*Pearson, K. (1900). Philosophical Magazine.*

$$\chi^2 = \sum_{i,j} \frac{(O_{ij} - E_{ij})^2}{E_{ij}}$$

Таблица сопряжённости (категориальный признак × метка). Слабая связь ($p \gg 0.05$) у признака, который по предметной логике должен быть предиктивным, является индикатором загрязнения меток: после массовой подмены статистическая зависимость между признаками и целевой переменной разрушается.

## 6. M8–M10: Методы на основе расстояний

### M8 — Расстояние Махаланобиса + MinCovDet

*Mahalanobis, P.C. (1936). Proceedings of the National Institute of Sciences of India; Rousseeuw & Van Driessen (1999). Technometrics, 41(3).*

$$D_M(x, \mu, \hat{\Sigma}) = \sqrt{(x - \mu)^T \hat{\Sigma}^{-1} (x - \mu)}$$

MinCovDet (Minimum Covariance Determinant) вычисляет оценку $\hat{\Sigma}$ по подмножеству 75% «наименее аномальных» точек, что обеспечивает точку разрушения 25% и гарантирует, что сами выбросы не искажают ковариационную матрицу. Статистический порог: $D_M^2 \sim \chi^2_p(0.975)$.

При высокой размерности матрицы признаков применяется PCA до $p \leq \lfloor n_c / 3 \rfloor$, чтобы выполнялось условие MinCovDet: $n_c \geq 2p + 2$.

**Почему этот метод незаменим:** единственный метод в блоке, учитывающий ковариационную структуру признаков. Стандартный Z-score смотрит на каждый признак независимо и пропускает многомерные выбросы, которые «нормальны» поканально, но аномальны в пространстве совместного распределения.

### M9 — KNN-согласованность меток

*Biggio, B. et al. (2011). Support Vector Machines Under Adversarial Label Noise. ACML; Paudice, A. et al. (2019). Label Sanitization against Label Flipping Poisoning Attacks. ECML-PKDD.*

Для каждого образца $x_i$:
$$\text{inconsistency}(x_i) = \frac{|\{j \in \text{kNN}(x_i) : y_j \neq y_i\}|}{k}$$

Наивный порог на абсолютное значение несогласованности создаёт зависимость от баланса классов: в задаче с соотношением 80/20 даже при случайных метках 20% соседей будут из другого класса. Поэтому метрика нормируется на примесь Джини:

$$\text{ratio} = \frac{\overline{\text{inconsistency}}}{\text{Gini}}, \quad \text{Gini} = 1 - \sum_c p_c^2$$

При $\text{ratio} = 0$ — идеальная кластеризация. При $\text{ratio} = 1$ — метки не лучше случайных. Порог блокировки: $\text{ratio} \geq 0.92$.

$k = 5$ из оригинальных статей (Biggio et al., Paudice et al.). Paudice et al. (2019) показали, что KNN-based очистка эффективна при уровне отравления < 30%.

**Почему этот метод незаменим:** наиболее чувствительный детектор подмены меток без референсного датасета. Использует прямое противоречие между признаками образца и его назначенной меткой.

### M10 — Обнаружение почти-дубликатов

*Huang, L. et al. (2011). Adversarial Machine Learning. AISec.*

Для каждого образца находится ближайший сосед. Пара признаётся конфликтующей, если расстояние меньше 5-го перцентиля распределения NN-расстояний И метки различаются. Используется выборка до 2000 образцов для контроля вычислительной сложности $O(n^2)$.

## 7. M11–M13: Плотностные детекторы

Три метода решают одну задачу — обнаружение выбросов — с разными инвариантами. Они не дублируют, а дополняют друг друга.

### M11 — Isolation Forest

*Liu, F.T., Ting, K.M., Zhou, Z.-H. (2008). Isolation Forest. ICDM; Liu et al. (2012). ACM TKDD, 6(1).*

Строится ансамбль случайных деревьев. Аномальная точка изолируется за меньшее число разбиений, чем нормальная. Оценка аномальности:

$$s(x, n) = 2^{-\frac{E[h(x)]}{c(n)}}, \quad c(n) = 2H(n-1) - \frac{2(n-1)}{n}$$

где $H$ — гармоническое число, $c(n)$ — ожидаемая глубина изоляции для случайной точки. Использование `contamination="auto"` устанавливает порог на $s = 0.5$, что соответствует ожидаемой глубине из оригинальной статьи. Это принципиально: фиксированный параметр `contamination` (например, 0.10) механически помечает ровно 10% точек при вызове `fit_predict`, что делает метрику нечувствительной к реальным данным.

### M12 — Local Outlier Factor

*Breunig, M.M. et al. (2000). LOF: Identifying Density-Based Local Outliers. SIGMOD. (> 7000 цитирований)*

$$\text{LOF}_k(x) = \frac{\sum_{o \in N_k(x)} \frac{\text{lrd}_k(o)}{\text{lrd}_k(x)}}{|N_k(x)|}, \quad \text{lrd}_k(x) = \frac{|N_k(x)|}{\sum_{o \in N_k(x)} \text{reach-dist}_k(x, o)}$$

$\text{LOF} = 1.0$ → точка такой же плотности, как соседи. $\text{LOF} > 1.5$ → точка значительно «одиночнее» своего окружения. Порог 1.5 из оригинальной статьи.

Используются сырые LOF-оценки из `negative_outlier_factor_` (а не бинарные предсказания с фиксированным `contamination`) — по той же причине, что и в M11.

**Принципиальное отличие от M11:** LOF работает локально и обнаруживает выбросы в неоднородных по плотности распределениях, где Isolation Forest ошибается.

### M13 — DBSCAN

*Ester, M. et al. (1996). A Density-Based Algorithm for Discovering Clusters in Large Spatial Databases with Noise. KDD; Schubert et al. (2017). ACM TODS, 42(3).*

Точки, не вошедшие ни в один плотный кластер (метка −1), — кандидаты в аномалии. $\varepsilon$ выбирается автоматически как 90-й перцентиль дистанций до $k$-го соседа (рекомендация из Ester et al. и Schubert et al.).

**Почему три детектора, а не один:** DBSCAN лучше работает при чётко выраженных кластерах; LOF — при размытых границах; IF — в высокой размерности. Атака, оптимизированная против одного детектора, неизбежно создаёт детектируемый сигнал для другого.

## 8. M14–M16: Спектральные и глобальные методы

### M14 — Spectral Signatures

*Tran, B., Li, J., Madry, A. (2018). Spectral Signatures in Backdoor Attacks. NeurIPS. arXiv:1811.00636.*

Для каждого класса $c$:

1. Центрирование: $\tilde{X}_c = X_c - \bar{X}_c$
2. SVD: $\tilde{X}_c = U \Sigma V^T$, берём $v_1$ — ведущий правый сингулярный вектор
3. Оценка аномальности: $r_i = (x_i^T v_1)^2$
4. Метрика: $\text{gap} = P_{95}(r) / P_{50}(r)$

**Теорема (Tran et al., 2018):** все известные бэкдор-атаки оставляют спектральный след — отравленные примеры имеют значительно большую проекцию на $v_1$, чем чистые.

Для центрированных гауссовых данных оценки $r_i$ следуют $\chi^2_1$-распределению, у которого $P_{95} / P_{50} \approx 3.84 / 0.455 \approx 8.4$. Порог $\text{gap} > 20$ установлен выше этого фонового значения.

### M15 — Robust PCA

*Candès, E.J., Li, X., Ma, Y., Wright, J. (2011). Robust Principal Component Analysis? JACM, 58(3).*

Разложение $X = L + S$, где $L$ — низкоранговая матрица «нормального» поведения, $S$ — разреженная матрица аномалий. Реализуется через PCA-реконструкцию: $\hat{X} = \text{PCA}(X)$, ошибка $\|x_i - \hat{x}_i\|_2$ — аппроксимация строки $S_i$.

Порог аномальности: $\text{median} + 3.5 \times \text{MAD}$ — критерий Iglewicz & Hoaglin, тот же что в M1. Единая методологическая база для обоих методов обеспечивает согласованность пороговых значений.

### M16 — Конформный тест сепарабельности

*Sánchez, A. et al. (2025). Provably effective detection of effective data poisoning attacks. arXiv:2501.11795.*

Мера несоответствия:

$$\text{score}(x_i) = \frac{d(x_i,\ \text{NN}_{\text{same}}(x_i))}{d(x_i,\ \text{NN}_{\text{diff}}(x_i)) + \varepsilon}$$

$\text{score} > 1$ означает, что точка ближе к чужому классу, чем к своему. Глобальная статистика $T = \frac{1}{n}\sum_i \mathbf{1}[\text{score}(x_i) > 1]$ сравнивается с распределением, полученным пермутационным тестом (300 пермутаций меток).

**Теорема (Sánchez et al., 2025):** если атака отравления достаточно эффективна против произвольного классификатора, то $T$ на отравленном датасете статистически значимо выше, чем на случайном расположении меток. Иными словами: эффективная атака обязательно создаёт детектируемое нарушение сепарабельности.

Уровень значимости $\alpha = 0.1$ (вместо классического 0.05) выбран намеренно для повышения чувствительности к умеренным нарушениям при приемлемом уровне ложных срабатываний. 

**Почему M16 незаменим:** это единственный метод в блоке с формальной теоретической гарантией. Все остальные методы — эвристики. M16 является финальным арбитром: если атака прошла M1–M15, теорема гарантирует её обнаружение при условии, что атака реально влияет на качество модели.

## 9. Матрица покрытия угроз

| Угроза | Покрывающие методы | Необходимость |
|---|---|---|
| Инъекция выбросов в признаки | NumericSentinel, M1, M8 | Без NumericSentinel ловится только на табличных данных; M8 нужен для многомерных выбросов |
| Подмена меток (label flip) | M9, M16, M2, M3 | M9 — наиболее чувствительный; M16 — теоретическая гарантия |
| Бэкдор-токены (backdoor) | SuspiciousToken, M14 | Оба обязательны: SuspiciousToken работает на сырых токенах, M14 — в пространстве признаков |
| Замена символов (homoglyph) | NonAscii | Единственный детектор этого класса атак |
| Дублирование с подменой метки | ExactLabelConflict, M10 | Разные пороги: ExactLabelConflict — точные совпадения, M10 — близкие |
| Изменение распределения признаков | M2, M3, M5, M6, M7 | Разные предположения о природе распределения — ни один не покрывает все случаи |

## 10. Пределы применимости

Без эталонного (чистого) датасета невозможно надёжно обнаружить:

**Умеренную подмену меток при сохранении баланса классов.** Если злоумышленник в равных долях переводит образцы из класса A в B и из B в A, баланс, распределения и KNN-согласованность меняются незначительно. M16 теоретически должен поймать такую атаку, однако при небольших объёмах выборки мощности пермутационного теста может не хватить.

**Атаки на уровень представленности класса при нормальном дисбалансе.** Если атака сводится к добавлению реалистичных примеров миноритарного класса, без структурных аномалий и нарушений сепарабельности, блок 2 её не обнаружит. Это задача для блока 3 (анализ производительности модели на валидационной выборке).

**PSI (Population Stability Index)** — метод M4, описанный в исходной документации — не реализован намеренно: он требует отдельного чистого референсного датасета и нарушает принцип автономности блока. При наличии механизма передачи референса PSI закрыл бы большинство из перечисленных пробелов.

## 11. Результаты тестирования

Блок протестирован на 11 датасетах финансовой и NLP-тематики с четырьмя вариантами каждого: один чистый и три отравленных разными методами. Результаты:

- **Чистые данные:** 0 ложных срабатываний из 11 (100% специфичность)
- **Отравленные данные:** 18 из 33 вариантов обнаружено (55% чувствительность)
- Не обнаруженные варианты: исключительно balanced label flip без структурных аномалий — предел метода без референса

Ложные срабатывания отсутствуют, что критично для production-среды: системный false positive приведёт к игнорированию сигналов или отказу от использования инструмента.
